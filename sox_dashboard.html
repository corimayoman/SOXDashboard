Context

What is SOX and why does it matter?

The Sarbanes-Oxley Act (SOX) requires publicly traded companies to maintain and demonstrate strong internal controls over financial reporting. For IT, this means proving that critical systems — like SAP, AWS, and databases — are properly managed, access is controlled, and changes are tracked.

Failing to execute these controls on time can result in audit findings, regulatory penalties, and reputational risk. This dashboard makes it easy to see the status of every control, every month, across every platform.

Status Indicators

What do the colors mean?

Every cell in the table represents one control in one month. The color tells you exactly what happened:

Executed

All good

The control was completed on time with evidence recorded.

⚠ Alert

Needs attention

The control is overdue. Action is required immediately.

Pending

Not run

The control was not executed in this period.

On Time

Upcoming

The due date hasn't passed yet. No action needed.

N/A

Not applicable

This control doesn't apply in this period (e.g. quarterly control in a non-quarter month).

Control Types

What types of controls are tracked?

Each control ID encodes its type in the third segment (e.g. GB.IT.ACC.SAP.01 → type is ACC). There are four types:

ACC

Access Controls

Who has access to what? Reviews of user accounts, roles, and privileges across all systems.

CHG

Change Controls

How are changes managed? Tracks configuration changes, transports, and system modifications.

OPE

Operations Controls

Is the system running correctly? Monitors batch jobs, data integrity, backups, and logs.

APD

Application/DB Controls

Database-level controls — admin access, HANA monitoring, and DB-specific security reviews.

Reading the Dashboard

How to use this dashboard in 5 steps

1

Check the summary cards at the top

Each card represents a month. Green means most controls executed. Pink means there are overdue controls that need attention. Yellow means the month is upcoming.

2

Look at the trend chart

The chart shows whether execution is improving or declining over time. A rising green line is good. A rising pink line means problems are accumulating.

3

Use filters to focus

Filter by Application (e.g. SAP, AWS), Control Type (ACC, CHG, OPE, APD), Frequency, or Year to narrow down to what matters to you.

4

Spot the red cells in the table

Any pink "⚠ Alert" cell is a control that missed its deadline. Click it to see the details: who owns it, when it was due, and any notes.

5

Click any cell for details

Every cell is clickable. The detail panel shows the control owner, due date, execution date, evidence link, and observations for that specific month.

FAQ

Common questions

What happens if a control shows "Alert"? ▼
An alert means the control was not executed before its due date. The responsible team should be contacted immediately. Persistent alerts may be flagged in the annual SOX audit.
How often is this data updated? ▼
Currently the data is manually updated by the IT controls team. The roadmap includes connecting directly to Google Sheets or Jira so updates happen automatically in real time.
What does "N/A" mean for a control? ▼
N/A means the control is not required in that specific month. For example, a quarterly control only needs to run once every three months — the other two months will show N/A. This is expected and not a problem.
Who is responsible for each control? ▼
Each control has an assigned owner shown in the "Owner" column and in the detail modal. Owners are typically the technical team responsible for that platform — e.g. Basis SAP, DBA Team, IST Architecture, GRC Team.
What is the difference between Monthly and Quarterly controls? ▼
Monthly controls must be executed every month. Quarterly controls only need to run once per quarter. Use the "Frequency" filter to see only the controls relevant to a given review cycle.
Can I export this data? ▼
Not yet — export functionality is on the roadmap. For now, use your browser's print function (Cmd+P / Ctrl+P) to save the current view as a PDF for reporting purposes.
